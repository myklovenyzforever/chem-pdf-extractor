# Codex for Open Source Application Draft

## 1. Describe your role: are you a primary or core maintainer?

I am the original author and primary maintainer of Chem-PDF-Extractor. I maintain the PDF-to-text workflow, configurable extraction fields, Windows one-click launcher, examples, documentation, tests, and release preparation.

## 2. Why does this repository qualify?

Chem-PDF-Extractor addresses a real research workflow: turning scientific PDF papers into structured literature data for chemical engineering, catalysis, materials, and environmental research. It is early-stage but useful for reproducible literature review and data-mining workflows.

## 3. How will you use API credits for your project?

I will use API credits to improve extraction prompts, build regression examples, test diverse PDF layouts, expand chemistry-oriented field templates, improve documentation, and support issue triage and release workflows without embedding user API keys.

## 4. Anything else we should know?

I am an AI + chemical engineering graduate student maintaining this project independently. The project is early-stage, avoids inflated impact claims, and focuses on safe, reproducible, permission-safe examples for research data extraction.
