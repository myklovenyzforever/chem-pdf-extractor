# Chem-PDF-Extractor 使用说明

## 1. 文件说明

常见文件包括：

- `ShuJuTiQuJiaoBen.py`：轻量入口脚本，用于启动 Web UI 或 CLI。
- `chem_pdf_extractor/`：主程序代码所在的 Python 包。
- `YiJianQiDong.bat`：Windows 一键启动脚本，通常用于打包版本。
- `YiLaiHuanJing/`：可选本地运行环境，适合不会配置 Python 的普通用户。
- `config.example.json`：配置模板，不含真实 API Key。
- `config.local.json`：用户自己的本地配置文件，不应上传 GitHub。

## 2. 启动方式

### Web UI

```powershell
python ShuJuTiQuJiaoBen.py
```

终端会显示本地网页地址。浏览器打开该地址即可使用。

### CLI

```powershell
python ShuJuTiQuJiaoBen.py --cli --pdf-mode pypdf_text
```

CLI 更适合调试、批处理或在 PyMuPDF 后端异常时使用稳定 fallback。

## 3. 模型配置

工具支持两类模型：

- 本地 Ollama 模型。
- OpenAI-compatible cloud API。

仓库不内置真实 API Key。你可以在网页的 LLM API 配置区域填写：

- LLM 服务名称。
- 模型名称。
- LLM API Key。
- LLM Base URL。

保存本地配置后，程序会生成 `config.local.json`。该文件已加入 `.gitignore`，只应保存在你自己的电脑上。

## 4. PDF 转换模式

当前支持多种 PDF 转 Markdown / 文本模式：

- `auto`
- `mineru`
- `pymupdf4llm`
- `pymupdf_text`
- `pypdf_text`

`pypdf_text` 是兼容性 fallback，通常更稳定，但对复杂表格、双栏排版、图文混排和扫描版 PDF 的解析能力较弱。复杂扫描版 PDF 可能需要 OCR 或 MinerU 支持。

## 5. 依赖安装

推荐 Python 3.11，以获得更好的 PDF 后端兼容性。

完整功能安装：

```powershell
python -m pip install -r requirements.txt
python ShuJuTiQuJiaoBen.py
```

Windows + Python 3.12 如遇 `pymupdf4llm` / `pymupdf` 安装或导入失败，可使用核心依赖安装：

```powershell
python -m pip install -r requirements-core.txt
python ShuJuTiQuJiaoBen.py --cli --pdf-mode pypdf_text
```

完整 PDF 后端也可通过可选依赖安装：

```powershell
python -m pip install ".[pdf]"
```

## 6. 处理 PDF

1. 在 “PDF 输入目录” 中填写论文 PDF 所在文件夹。
2. 选择 PDF 转 Markdown / 文本方式。
3. 配置抽取字段。
4. 点击开始处理。
5. 查看输出文件和日志。

建议先用 3-5 篇 PDF 测试，再处理上百篇或上千篇。

## 7. 暂停、继续和断点续跑

- 暂停任务：保存已有结果，方便稍后继续。
- 继续任务：复用已有 `md文件/`、`抽取缓存/` 和断点结果。
- 停止任务：停止当前处理流程。

## 8. 输出文件

常见输出包括：

- `提取结果.xlsx`：最终有效结果。
- `错误日志.txt`：错误原因和失败环节。
- `坏数据.xlsx/jsonl`：填写率太低而删除的数据。
- `可疑数据.xlsx/jsonl`：保留但需要人工复核的数据。
- `md文件/`：PDF 转出的 Markdown 和图片。
- `抽取缓存/`：大模型抽取缓存。
- `提取失败源文件/`：失败 PDF 的副本。

## 9. 注意事项

- AI 抽取结果必须人工复核，不能直接作为最终科研结论。
- 不要把真实论文 PDF、真实 API Key、token、`.env` 或 `config.local.json` 上传 GitHub。
- 使用云端 API 可能产生费用。
- 不同 PDF 的排版复杂度不同，抽取质量需要人工检查。
