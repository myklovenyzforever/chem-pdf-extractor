# Chem-PDF-Extractor 使用说明

## 1. 文件说明

常见文件包括：

- `ShuJuTiQuJiaoBen.py`：主程序。
- `YiJianQiDong.bat`：Windows 一键启动脚本。
- `YiLaiHuanJing/`：可选本地运行环境，适合不会配置 Python 的普通用户。
- `config.example.json`：配置模板，不含真实 API Key。
- `config.local.json`：用户自己的本地配置文件，不应上传 GitHub。

## 2. 一键启动

如果你下载的是带 `YiLaiHuanJing/` 的 Release 包，直接双击：

```text
YiJianQiDong.bat
```

终端会显示本地网页地址。浏览器打开该地址即可使用。

## 3. 填写 API Key

仓库不内置真实 API Key。你可以在网页的 LLM API 配置区域填写：

- LLM 服务名称
- 模型名称
- LLM API KEY
- LLM BASE URL

填写后可以点击“保存本地配置”，程序会生成 `config.local.json`。这个文件只保存在你电脑上，已加入 `.gitignore`。

## 4. 处理 PDF

1. 在“PDF 输入目录”中填写论文 PDF 所在文件夹。
2. 选择 PDF 转 Markdown / 文本方式。
3. 确认抽取字段。
4. 点击“开始处理”。

建议先用 3-5 篇 PDF 测试，再处理上百篇或上千篇。

## 5. 暂停和继续

- 暂停任务：保存已有结果，方便稍后继续。
- 继续任务：复用已有 `md文件/`、`抽取缓存/` 和断点结果。
- 停止任务：停止当前处理流程。

## 6. 输出文件

常见输出：

- `提取结果.xlsx`：最终有效结果。
- `错误日志.txt`：错误原因和失败环节。
- `坏数据.xlsx/jsonl`：填写率太低而删除的数据。
- `可疑数据.xlsx/jsonl`：保留但需要人工复核的数据。
- `md文件/`：PDF 转出的 Markdown 和图片。
- `抽取缓存/`：大模型抽取缓存。
- `提取失败源文件/`：失败 PDF 的副本。

## 7. 注意事项

- 大模型结果必须人工复核，不能直接作为最终科研结论。
- 不要把论文 PDF、真实 API Key、`config.local.json` 上传 GitHub。
- 使用云端 API 会产生费用。
- MinerU 质量较好但可能较慢。
